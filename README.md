# Описание репозитория **RustWorks**

Данный репозиторий содержит проекты, реализованные на языке программирования Rust.

## Проекты

### 1. [Текстовый редактор им. Жмыха Ящерицы](https://github.com/MrFandore/RustWorks/tree/main/TekstRedactor)

**Цель:** Разработка полнофункционального текстового редактора с современным пользовательским интерфейсом и расширенными возможностями редактирования.

#### Основной функционал:
- **Многодокументный интерфейс** - работа с несколькими файлами одновременно через систему вкладок
- **Расширенная система отмены/повтора** - до 50 шагов истории изменений
- **Интеллектуальный поиск и замена** - с поддержкой регистра и целых слов
- **Подробная статистика документа** - подсчет страниц, слов, символов, строк и абзацев
- **Автоматическое сохранение** - защита от потери данных
- **Настраиваемый интерфейс** - светлая и темная темы оформления
- **Горячие клавиши** - быстрое выполнение операций
- **Поддержка различных форматов** - текстовые файлы, Markdown, исходный код

#### Технические особенности:
- **Язык программирования:** Rust
- **Графический фреймворк:** egui/eframe
- **Архитектура:** Модель-Представление с четким разделением ответственности
- **Кроссплатформенность:** Windows, Linux, macOS

---

### 2. [Система мониторинга процессов компьютера "Справедливый Абрикос"](https://github.com/MrFandore/RustWorks/tree/main/MonitorSystemOPs)

**Цель:** Разработка системы мониторинга системных ресурсов компьютера в реальном времени с веб-интерфейсом и функционированием в качестве службы Windows.

#### Основной функционал:
- **Мониторинг системных ресурсов** - отслеживание использования CPU, памяти, дискового пространства и сетевой активности
- **Веб-интерфейс в реальном времени** - отображение метрик с автоматическим обновлением каждые 3 секунды
- **Служба Windows** - автоматический запуск при старте системы и управление через CLI
- **Система уведомлений** - оповещения о критических состояниях системы (высокая загрузка CPU, памяти, диска)
- **Защита конфигурации** - шифрование настроек и проверка прав доступа

#### Технические особенности:
- **Язык программирования:** Rust
- **Веб-сервер:** Warp (асинхронный)
- **Сбор метрик:** PowerShell команды и Windows API
- **Архитектура:** Асинхронная обработка с использованием Tokio, модульная структура
- **Управление службой:** Интеграция с Windows Service Manager

### 3. [Многопользовательский чат "MassagerMAX"](https://github.com/MrFandore/RustWorks/tree/main/MassagerMAX)

**Цель:** Разработка многопользовательского консольного чата с клиент-серверной архитектурой, системой авторизации и поддержкой приватных сообщений.

#### Основной функционал:
- **Многопользовательский сервер** - одновременная обработка подключений от нескольких клиентов
- **Система авторизации и регистрации** - безопасное хранение паролей с хэшированием bcrypt
- **Приватные сообщения** - личная переписка между пользователями через команду `/whisper`
- **Журналирование сообщений** - сохранение всей истории чата в файл для последующего анализа
- **Интуитивный командный интерфейс** - понятные команды на русском языке
- **Многопоточная архитектура** - обработка каждого клиента в отдельном потоке

#### Технические особенности:
- **Язык программирования:** Rust
- **Сетевое взаимодействие:** Стандартная библиотека (std::net)
- **Синхронизация:** Arc<Mutex<...>> для потокобезопасного доступа
- **Безопасность:** Хэширование паролей с использованием bcrypt
- **Хранение данных:** JSON-файлы для пользователей и логов
- **Архитектура:** Клиент-серверная модель с блокирующим I/O

#### Команды чата:
- `/register <имя> <пароль>` - регистрация нового пользователя
- `/login <имя> <пароль>` - вход в систему
- `/msg <сообщение>` - отправка сообщения всем пользователям
- `/whisper <имя> <сообщение>` - отправка приватного сообщения
- `/exit` - выход из чата

#### Особенности реализации:
- Минимальные зависимости (только serde_json, serde и bcrypt)
- Консольный интерфейс с русскоязычными подсказками
- Поддержка русского текста в сообщениях
- Автоматическое создание файлов данных при первом запуске
- Простая сборка и запуск без сложных настроек

## Требования для запуска проектов

Для работы с проектами необходимо:
1. Установленный Rust (версии 1.70 или выше)
2. Установленный Cargo (менеджер пакетов Rust)
3. Для проекта мониторинга: Windows 10/11 (для службы) и PowerShell
4. Минимум 512 MB оперативной памяти
5. 10 MB свободного дискового пространства на проект

### Рекомендуемые требования:
- 1 GB оперативной памяти
- Многоядерный процессор
- 1.5 GB свободного дискового пространства на проект

## Как использовать

### Установка и запуск:

1. **Клонируйте репозиторий:**
```bash
git clone https://github.com/MrFandore/RustWorks.git
cd RustWorks
```

2. **Перейдите в директорию проекта и соберите его:**
```bash
# Для текстового редактора
cd TekstRedactor
cargo build --release

# Для системы мониторинга
cd MonitorSystemOPs
cargo build --release

# Для чата MassagerMAX
cd MassagerMAX
cargo build --release
```

3. **Запустите приложение:**
```bash
# Текстовый редактор
cargo run --release

# Система мониторинга (консольный режим)
cargo run --release

# Чат MassagerMAX (сначала сервер, потом клиенты)
cargo run --release --bin server
cargo run --release --bin client
```

### Управление службой мониторинга (требует прав администратора):
```bash
# Установка службы
MonitorSystemOPs.exe install

# Запуск службы
MonitorSystemOPs.exe start

# Веб-интерфейс будет доступен по http://127.0.0.1:8080
```

### Запуск чата MassagerMAX:
```bash
# В первом терминале запустите сервер
cargo run --release --bin server

# Во втором терминале запустите первого клиента
cargo run --release --bin client

# В третьем терминале (опционально) запустите второго клиента
cargo run --release --bin client
```

## Технологический стек

### Общий для проектов:
- **Язык программирования:** Rust 1.70+
- **Система сборки:** Cargo
- **Обработка ошибок:** anyhow (в некоторых проектах)

### Для текстового редактора:
- **Графический интерфейс:** egui/eframe
- **Обработка изображений:** image (для иконки)
- **Диалоговые окна:** rfd
- **Сериализация:** serde

### Для системы мониторинга:
- **Веб-сервер:** warp
- **Асинхронная обработка:** tokio
- **Работа со временем:** chrono
- **Управление службами Windows:** windows-service
- **Шифрование:** base64

### Для чата MassagerMAX:
- **Сетевое взаимодействие:** Стандартная библиотека Rust (std::net)
- **Сериализация:** serde, serde_json
- **Хэширование паролей:** bcrypt
- **Многопоточность:** Стандартная библиотека (std::thread)
- **Синхронизация:** std::sync::{Arc, Mutex}

## Разработка

Для разработки и модификации проектов:

1. Установите Rust и Cargo
2. Клонируйте репозиторий
3. Используйте `cargo run` для запуска в режиме разработки
4. Для сборки релизной версии используйте `cargo build --release`

### Сборка для различных платформ:

#### Текстовый редактор:
```bash
# Windows
cargo build --release --target x86_64-pc-windows-msvc

# Linux
cargo build --release --target x86_64-unknown-linux-gnu

# macOS
cargo build --release --target x86_64-apple-darwin
```

#### Чат MassagerMAX (кроссплатформенный):
```bash
# Проект использует только стандартную библиотеку Rust,
# поэтому сборка для разных платформ не требует специальных настроек

# Windows
cargo build --release --bin server
cargo build --release --bin client

# Linux/macOS
cargo build --release --bin server
cargo build --release --bin client
```

## Особенности реализации

### Текстовый редактор:
- **Безопасность:** Полное отсутствие неопределенного поведения благодаря системе владения Rust
- **Производительность:** Нативная компиляция и эффективные алгоритмы работы с текстом
- **Кроссплатформенность:** Единая кодовая база для всех поддерживаемых ОС
- **Доступность:** Интуитивно понятный интерфейс с поддержкой горячих клавиш

### Система мониторинга:
- **Надежность:** Использование системных API для точного сбора метрик
- **Производительность:** Асинхронная обработка запросов и эффективное хранение данных
- **Безопасность:** Проверка прав доступа и шифрование конфигурации
- **Масштабируемость:** Модульная архитектура позволяет легко добавлять новые метрики

### Чат MassagerMAX:
- **Простота:** Минимальное количество зависимостей, использование стандартной библиотеки
- **Безопасность:** Хэширование паролей с использованием современного алгоритма bcrypt
- **Масштабируемость:** Многопоточная архитектура позволяет обрабатывать множество клиентов
- **Надежность:** Блокирующий I/O обеспечивает стабильную работу даже при высокой нагрузке
- **Удобство:** Русскоязычный интерфейс с понятными командами для пользователей

## Цели проектов

Каждый проект в репозитории решает конкретные учебные и практические задачи:

1. **Текстовый редактор:** Изучение графических интерфейсов на Rust, работа с файловой системой, реализация сложного UI
2. **Система мониторинга:** Освоение работы с системными API, создание веб-серверов, разработка служб Windows
3. **Многопользовательский чат:** Изучение сетевого программирования, многопоточности, клиент-серверной архитектуры
